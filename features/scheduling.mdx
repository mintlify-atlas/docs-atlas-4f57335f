## Overview

The scheduling system organizes pottery classes into **turnos** (shifts) - recurring time slots on specific days of the week. Each student is assigned to one turno, which determines their regular class attendance schedule.

## Turno Model

The `Turno` model defines weekly class schedules:

```prisma
model Turno {
  id         String   @id @default(dbgenerated("gen_random_uuid()"))
  created_at DateTime @default(now())
  dia        String   // e.g., "Lunes", "Martes"
  horario    String   // e.g., "14:00 - 17:00"
  alumnas    Alumna[] // Inverse relation to students
}
```

| Field | Type | Description |
|-------|------|-------------|
| `dia` | String | Day of the week in Spanish (Lunes, Martes, Miércoles, etc.) |
| `horario` | String | Time range in 24-hour format (e.g., "18:00 - 20:00") |
| `alumnas` | Relation | List of students assigned to this shift |

<Note>
  Turnos are **template schedules**, not individual class instances. They define recurring weekly time slots.
</Note>

## Admin Workflows

### Creating a New Turno

Admins use the **GestionTurnos** component to create class schedules:

<Steps>
  <Step title="Select day of week">
    Choose from dropdown: Lunes, Martes, Miércoles, Jueves, Viernes, Sábado, Domingo
  </Step>
  
  <Step title="Enter time range">
    Type the class hours (e.g., "14:00 - 17:00", "18:00 a 20:00")
  </Step>
  
  <Step title="Click 'Agregar +'">
    The new turno is created and appears in the active shifts grid
  </Step>
</Steps>

**Example turnos**:
- Lunes 14:00 - 17:00 (afternoon session)
- Miércoles 18:00 - 20:00 (evening session)
- Sábado 10:00 - 13:00 (weekend morning)

### Editing Existing Turnos

Admins can modify shift schedules:

1. **Click the edit icon** (✏️) on a turno card
2. The form switches to edit mode with:
   - Blue border highlighting
   - Pre-filled current values
   - "Guardar Cambios" button
3. **Modify** day or time as needed
4. **Save** to update the turno

<Info>
  Editing a turno automatically updates the schedule for all students assigned to it. The change is reflected immediately in student profiles.
</Info>

### Deleting Turnos

Admins can remove unused shifts:

- Click the delete icon (🗑️) on a turno card
- Confirm deletion in the popup

<Warning>
  **Cannot delete turnos with assigned students**. The database foreign key constraint prevents deletion if any `Alumna` records reference this `turno_id`. You'll see an error message: "No se puede borrar: Probablemente hay alumnas asignadas a este turno."
</Warning>

To delete a turno with students:
1. First reassign all students to different turnos
2. Then delete the empty turno

## Student Assignment

### Assigning During Registration

When creating a new student, admins can assign a turno:

- **Dropdown menu** in student creation form
- Shows all available turnos as "Día - Horario"
- Select one or leave as "-- Sin Turno --"

The assignment creates the foreign key relationship:

```prisma
model Alumna {
  turno_id String? @db.Uuid
  turno    Turno?  @relation(...)
}
```

### Changing Student Turno

Admins can reassign students to different shifts:

1. **Edit the student** profile
2. **Select new turno** from dropdown
3. **Save changes**

The student's schedule updates immediately.

### Students Without Turnos

Students can exist without an assigned turno (`turno_id: null`):

- Shows "Sin turno" badge in student management
- Useful for:
  - Students on hold/hiatus
  - Open studio members without fixed schedule
  - Trial/guest students

## Student View

Students see their assigned turno in their profile view:

- **Display format**: Blue badge with day and time
- **Example**: "📅 Lunes 14:00 - 17:00"
- **Location**: Shown on student profile card in admin view and in student's own profile section

<Info>
  Students cannot change their own turno - this is admin-controlled to manage class capacity and organization.
</Info>

## API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/turnos` | Admin | Get all turnos |
| POST | `/turnos` | Admin | Create a new turno |
| PUT | `/turnos/:id` | Admin | Update turno details |
| DELETE | `/turnos/:id` | Admin | Delete turno (fails if students assigned) |

### Create Turno Request

```json
{
  "dia": "Lunes",
  "horario": "14:00 - 17:00"
}
```

Both fields are required. The endpoint returns 400 error if either is missing:

```javascript
if (!dia || !horario) {
  return res.status(400).json({ 
    error: 'Día y horario son obligatorios' 
  })
}
```

### Update Turno Request

```json
{
  "dia": "Martes",           // optional
  "horario": "15:00 - 18:00"  // optional
}
```

Partial updates are supported - include only fields to change.

## Database Relationships

The turno-alumna relationship uses optional foreign key:

```prisma
model Alumna {
  turno_id String? @db.Uuid
  turno    Turno?  @relation(
    fields: [turno_id], 
    references: [id], 
    onUpdate: NoAction
  )
}
```

- **Optional**: `turno_id` can be null (students without schedule)
- **On Update**: NoAction (prevents accidental cascade updates)
- **On Delete**: Not specified, so default RESTRICT prevents deleting turno with assigned students

## Form Validation

The turno management form includes:

- **Day selector**: Dropdown with all 7 days of week
- **Time input**: Free-form text field (flexible format)
- **Submit disabled**: While loading/saving to prevent duplicates
- **Instant feedback**: Success (✅) or error (❌) messages appear below form

### Accepted Time Formats

The `horario` field accepts various formats:

- `14:00 - 17:00` (with dash)
- `18:00 a 20:00` (with 'a')
- `10:00 - 13:00hs` (with 'hs' suffix)

No strict validation - admins can use workshop-specific conventions.
