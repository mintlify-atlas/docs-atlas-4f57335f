## Overview

The notification system allows administrators to send messages to individual students. Notifications appear in the student's app with unread indicators and can be marked as read or deleted.

## Notification Model

The `Notificacion` model stores messages sent to students:

```prisma
model Notificacion {
  id         BigInt   @id @default(autoincrement())
  mensaje    String
  leida      Boolean  @default(false)
  created_at DateTime @default(now())
  alumna_id  String   @db.Uuid
  alumna     Alumna   @relation(
    fields: [alumna_id], 
    references: [id], 
    onDelete: Cascade
  )
}
```

| Field | Type | Description |
|-------|------|-------------|
| `mensaje` | String | Notification content/text |
| `leida` | Boolean | Read status (default: false) |
| `created_at` | DateTime | Timestamp when notification was sent |
| `alumna_id` | UUID | Student who receives this notification |

<Note>
  Each notification is linked to exactly one student. To message multiple students, create separate notification records for each.
</Note>

## Admin Workflows

### Creating Notifications

Administrators create notifications through the backend API:

**Endpoint**: `POST /notificaciones` (implementation varies)

**Request body**:
```json
{
  "alumna_id": "uuid-of-student",
  "mensaje": "Tu pieza está lista para retirar. Pasa por el taller!"
}
```

The notification is created with:
- `leida: false` (unread by default)
- `created_at: now()` (current timestamp)

The bell icon shows a visual badge when there are unread notifications:


- **Red dot** (terracotta `#84572F`) appears on bell icon
- Calculated by: `notificaciones.some(n => !n.leida)`

## Notification Actions

### Mark All as Read

When the notification panel is opened:

```javascript
onClick={() => {
  setMostrarNotificaciones(true)
  if (tieneNotificacionesNuevas) {
    api.put('/alumnas/me/notificaciones/leer')
    setNotificaciones(prev => 
      prev.map(n => ({ ...n, leida: true }))
    )
  }
}}
```

**Endpoint**: `PUT /alumnas/me/notificaciones/leer`

- Marks all notifications for current student as `leida: true`
- Updates local state immediately for responsive UI
- Red dot indicator disappears

### Delete All Notifications

Students can clear their notification history:

1. **Click "Limpiar todas"** button in panel header
2. Notifications are deleted from database
3. Panel shows empty state

**Endpoint**: `DELETE /alumnas/me/notificaciones`

```javascript
await api.delete('/alumnas/me/notificaciones')
setNotificaciones([])
```

<Warning>
  Deleting notifications is permanent - they cannot be recovered. Students may want to read important messages before clearing.
</Warning>

## API Endpoints

### Get Student Notifications

**Endpoint**: `GET /alumnas/me/notificaciones`

**Auth**: Student (JWT token)

**Response**:
```json
[
  {
    "id": 1,
    "mensaje": "Tu pieza está lista para retirar",
    "leida": false,
    "created_at": "2024-03-15T14:30:00Z",
    "alumna_id": "uuid-string"
  }
]
```

### Mark Notifications as Read

**Endpoint**: `PUT /alumnas/me/notificaciones/leer`

**Auth**: Student

**Action**: Sets `leida: true` for all notifications belonging to the student

### Delete All Notifications

**Endpoint**: `DELETE /alumnas/me/notificaciones`

**Auth**: Student

**Action**: Permanently deletes all notifications for the student

### Generic Endpoints

The generic notification endpoints (admin access):

**GET `/notificaciones`**: Get notifications for a specific student

**POST `/notificaciones/:id/read`**: Mark individual notification as read

These are defined in `notificaciones.controller.js` but not currently exposed in the frontend UI.

## Data Access Control

Notifications are filtered by user:

```javascript
export const getMisNotificaciones = async (req, res, next) => {
  const serialized = await getMisNotificacionesService(req.user.id)
  res.json(serialized)
}
```

- **Students**: Only see their own notifications (filtered by `alumna_id`)
- **Admins**: Can access all notifications or filter by student

<Info>
  The service layer ensures students can never access notifications belonging to other students.
</Info>

## Cascade Deletion

When a student account is deleted:

```prisma
alumna Alumna @relation(..., onDelete: Cascade)
```

All notifications for that student are automatically removed to maintain database integrity.

## BigInt Serialization

Notification IDs use BigInt (autoincrement). The controller includes:

```javascript
BigInt.prototype.toJSON = function () {
  return this.toString()
}
```

This converts notification IDs to strings in JSON responses.
